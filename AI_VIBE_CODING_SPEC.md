# eGadget Web Simulator - 仕様書 (AIバイブコーディング用)

## 1. プロジェクト概要 (Project Overview)
本プロジェクトは、教育・実験用のロボット（eGadget）をブラウザ上でシミュレーションするためのWEBアプリケーションです。
フロントエンドはReactベースで構築され、ユーザーが記述した「C言語ライクなロボット制御コード」をブラウザ上でJavaScriptのGenerator関数に変換（トランスパイル）し、HTML5 Canvas上で「2輪差動駆動型ロボット」の動きやセンサーの反応をシミュレートします。

## 2. 技術スタック (Tech Stack)
- **フロントエンド:** React 19, Vite, Tailwind CSS (v4)
- **バックエンド:** FastAPI (Python) - 初期設定のみ完了（CORS設定済み）
- **主要な技術要素:** HTML5 Canvas (2D描画), JavaScript Generators (同期・ブロッキング処理のエミュレーション)

## 3. ディレクトリ構造と主要ファイル (Structure & Key Files)

### バックエンド (`backend/`)
- `main.py`: FastAPIのメインサーバーファイル。現在はスタブ状態ですが、今後DB接続やAPI処理、実機ロボットとの連携が必要になった際の基盤となります。

### フロントエンド (`frontend/`)
- `src/App.jsx`: シミュレータのメインUIコンポーネント。Canvasの描画ループ（`requestAnimationFrame`）、コースの図形編集・配置UI、シミュレーションの状態（再生・停止・初期化）管理を統括しています。
- `src/config/robotConfig.js`: ロボットの機体サイズ、車輪の幅、最高速度、カラーセンサーの配置（id: 2=Center, 5=Left, 6=Rightに対応するCN2, CN5, CN6）などを定義する設定ファイル。
- **コアモジュール (`src/utils/`)**:
  - `transpiler.js`: ユーザが記述したC言語風のコードをJSのGenerator関数へ変換します。`while(TRUE)` の無限ループや、`wait_ms()` による待機処理を `yield` に置換することで、ブラウザをフリーズさせずにステップ実行する仕組みを提供します。（正規表現ベースの簡易パーサを採用）
  - `physics.js`: 左右モーターの入力値からロボットの2次元座標（x, y）と角度（rotation）を更新する物理演算モジュールです（2輪差動駆動型のキネマティクス）。
  - `sensors.js`: シミュレータ内のロボットの座標・角度に基づいて各センサーの現在位置を計算し、Canvasからピクセルデータ（`getImageData`）を読み取ってコース上の「色（ライン）」を検知します。
  - `renderer.js`: Canvasへコースのグラフィック（矩形・楕円・アップロード画像）およびロボットの機体・車輪・センサーを描画します。
  - `editorHelpers.js`: 画面上のコース作成ツール（マウスでの図形選択、移動、リサイズ）に関する当たり判定などのヘルパー関数群です。

## 4. シミュレーションの実行モデル (Simulation Execution Model)
1. **コード変換フェーズ**: ユーザが書いたCコード（`user_main`関数等）が `transpiler.js` によってGenerator関数に変換され、実行用のインスタンスが生成されます。
2. **メインループ (`App.jsx` 内の `animate()`)**:
   - 描画 (`app.drawScene()`): コース全体の描画 → センサー読み取り → ロボットの描画 の順に行われます。
   - センサー更新: `sensors.js` を利用し、ロボット直下のCanvasのピクセル色を元に `CN2`, `CN5`, `CN6` などの判定結果を取得します。
   - コード実行 (`yield`): Generatorの `.next()` を呼び出し、トランスパイルされたユーザコードを1フレーム分実行します（変数代入や関数呼び出しなど）。
   - アクチュエータ反映: ユーザコード内に `motor(l, r)` の呼び出しが含まれていた場合、フロントエンドのステート（モーター値）が更新されます。
   - 物理演算: `physics.js` を実行し、モーター値に基づきロボットを移動させます。

## 5. 今後の拡張・改修時のAIへの指示（Vibe Coding Guidelines）
今後、AI（あなた）がこのプロジェクトのコードを読んだり改修したりする際は以下の点に留意してください。

- **責務の分離**: `App.jsx` が巨大になりがちなので、UIイベント・描画・ロジックが混在しないよう、`src/utils/` にあるモジュールの役割分担を崩さずに拡張してください。
- **UI/UX**: スタイリングには Tailwind CSS を使用しています。既存のクラスを利用し、モダンで直感的なインターフェースを心がけてください。
- **パフォーマンス**: Canvasの再描画ループ (`requestAnimationFrame`) 内での処理はパフォーマンスに直結します。描画ループ内で重い処理（不要なDOM操作や複雑な計算など）を行わないようにしてください。
- **トランスパイラの仕様**: `transpiler.js` は現在、簡単な正規表現でCコードをJSに変換しています。C言語の複雑な構文（ポインタ、構造体など）のサポートを追加する場合は、正規表現の限界を考慮し、必要であればより堅牢なASTパーサ（例: Tree-sitter や手書きの再帰下降パーサ）への移行を提案してください。
- **API通信拡張**: もし今後、本シミュレータと実機との連携や、ユーザーデータの保存処理が必要になった場合は、フロントエンドの機能追加にとどまらず、適宜 `backend/main.py` に新しいエンドポイントを作成してCORS越しに連携させてください。
